Torrent Automator Master Guide

This comprehensive guide walks you through installing Python, setting up background directory monitoring, creating desktop controllers with native notifications, and configuring automated space cleanup with safety locks.

Part 1: Installing Python (From Scratch)

Before configuring the scripts, Python must be correctly set up on your system.

1. Download the Installer

  • Go to the official website: python.org/downloads
  • Click the yellow Download Python 3.x.x button.

2. Crucial Steps During Installation

  • Open the downloaded installer file.
  • CRITICAL: At the very bottom of the first window, check the box that says "Add python.exe to PATH". If you skip this, your laptop won't recognize the Python command environment.
  • Click Install Now and let it finish.
  • THE FINAL GUARD: Once the installation finishes successfully, look at the very bottom of the closing window. Click the option that says "Disable path length limit". This ensures long torrent folder names or deeply nested media files won't break your automated transfers down the road.

Part 2: Setting Up the File Paths

Create a folder on your computer named D:\Scripts. This is where all configuration scripts, source files, and automated activity logs will live.

Part 3: The 3 Core Files

You need to create three separate files. Open Notepad for each, paste the respective code blocks, and save them exactly as named below.

File 1: The Automation Engine (D:\Scripts\torrent_watcher.py)

Save this file with a .py extension inside your script folder. This code features an advanced Auto-Delete Pipeline equipped with a byte-size verification safety lock to safeguard your files against data loss.

import os
import shutil
import time
import subprocess
from datetime import datetime

# --- CONFIGURATION ---
WATCH_DIRECTORY = r"D:\Path\To\Your\Utorrent\Downloads"  # Where µTorrent saves finished files
TARGET_DIRECTORY = r"E:\Downloaded_Torrents"             # Your external drive destination
LOG_FILE = r"D:\Scripts\watcher_log.txt"
# ---------------------

def log_message(message):
    timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    print(f"[{timestamp}] {message}")
    with open(LOG_FILE, "a", encoding="utf-8") as f:
        f.write(f"[{timestamp}] {message}\n")

def send_windows_notification(title, message):
    """Triggers a native Windows desktop notification banner."""
    powershell_code = f'''
    [void] [System.Reflection.Assembly]::LoadWithPartialName("System.Windows.Forms");
    $notification = New-Object System.Windows.Forms.NotifyIcon;
    $notification.Icon = [System.Drawing.SystemIcons]::Information;
    $notification.BalloonTipIcon = "Info";
    $notification.BalloonTipTitle = "{title}";
    $notification.BalloonTipText = "{message}";
    $notification.Visible = $true;
    $notification.ShowBalloonTip(5000);
    '''
    try:
        subprocess.run(["powershell", "-Command", powershell_code], capture_output=True, text=True)
    except Exception as e:
        log_message(f"Failed to send notification: {str(e)}")

def get_dir_size(path):
    """Calculates total folder or file size to see if it is stable."""
    if os.path.isfile(path):
        return os.path.getsize(path)
    total_size = 0
    for dirpath, _, filenames in os.walk(path):
        for f in filenames:
            fp = os.path.join(dirpath, f)
            if os.path.exists(fp):
                total_size += os.path.getsize(fp)
    return total_size

def main():
    log_message("Watcher service started with Auto-Delete enabled. Monitoring downloads...")
    copied_items = set()

    if os.path.exists(WATCH_DIRECTORY):
        copied_items = set(os.listdir(WATCH_DIRECTORY))

    while True:
        try:
            if not os.path.exists(WATCH_DIRECTORY) or not os.path.exists(TARGET_DIRECTORY):
                time.sleep(10)
                continue

            current_items = os.listdir(WATCH_DIRECTORY)

            for item in current_items:
                if item.endswith(".!ut") or item in copied_items:
                    continue

                source_path = os.path.join(WATCH_DIRECTORY, item)
                destination_path = os.path.join(TARGET_DIRECTORY, item)

                size_before = get_dir_size(source_path)
                time.sleep(5)
                size_after = get_dir_size(source_path)

                if size_before == size_after and size_before > 0:
                    log_message(f"Detected completed item: {item}. Starting copy...")
                    
                    send_windows_notification("Torrent Download Complete", f"Copying '{item}' to external drive...")
                    
                    if os.path.isdir(source_path):
                        shutil.copytree(source_path, destination_path, dirs_exist_ok=True)
                    else:
                        shutil.copy2(source_path, destination_path)
                        
                    if os.path.exists(destination_path):
                        source_final_size = get_dir_size(source_path)
                        dest_final_size = get_dir_size(destination_path)
                        
                        if source_final_size == dest_final_size:
                            log_message(f"Size verification passed for '{item}' ({dest_final_size} bytes). Deleting local copy...")
                            
                            if os.path.isdir(source_path):
                                shutil.rmtree(source_path)
                            else:
                                os.remove(source_path)
                                
                            log_message(f"Successfully deleted original: {item}")
                            send_windows_notification("Transfer & Cleanup Successful", f"'{item}' is secure on external storage. Local space cleared.")
                        else:
                            log_message(f"CRITICAL SIZE MISMATCH: '{item}' source is {source_final_size} bytes, but destination is {dest_final_size} bytes. Deletion aborted for data safety.")
                            send_windows_notification("Transfer Warning", f"Size mismatch detected for '{item}'. Original file preserved for safety.")
                    else:
                        log_message(f"CRITICAL ERROR: Destination path for '{item}' does not exist. Deletion skipped.")
                    
                    copied_items.add(item)

        except Exception as e:
            log_message(f"Error encountered: {str(e)}")

        time.sleep(10)

if __name__ == "__main__":
    main()

Part 5: Notification System Breakdown

  • Service Started (Blue Info Icon): Triggered when the script boots up manually or at startup. Confirms that monitoring is active.
  • Copying Initiated (Information Icon): Dispatched the moment a download hits 100%. Lets you know the file transfer to your external drive has begun.
  • Transfer & Cleanup Successful (Success Check): Alerts you as soon as the copy safely finishes and passes the file size test. Confirms your data is intact on the external disk and the original has been purged to save local space.
  • Service Stopped (Yellow Warning Icon): Shows when the script execution is completely stopped via the controller file, verifying that background processing resources are cleared.

Part 6: Smart Scenarios Handled by Your Automator

1. Zero Resource Drain (0% CPU Sleep Guard)

While monitoring directories, the thread handles an explicit 10-second idle frame. Windows automatically registers it inside a suspended execution stack, consuming 0% processor assets and minimal working memory profiles (~12MB RAM), keeping system performance snappy.

2. Missing External Drive Protection

If the targeted backup drive is detached, the program avoids crashing out with runtime fatal exceptions. Structural assertions detect the folder availability flags, break loop mechanics cleanly, and monitor path presence states patiently every 10 seconds without generating massive error logs.

3. Duplicate Prevention Across System Restarts

Volatile memory structures lose tracking variables upon reboots. To prevent historical library items from re-copying to external paths over again, the engine executes an array initialization pass upon boot, caching old structures so it only targets newly initialized downloads.

4. Automatic Space-Cleaning Integrity Lock

The system never calls a destructive deletion sequence blindly. It measures the byte profile of the copied folder/file on your external drive and maps it against the local source file size. If the values differ by even a single byte—implying a bad transfer or corrupted disk save—the automation halts the cleanup sequence, logs a warning, and retains the original download safely on your machine.

5. Invisible UI Execution with Native Desktop HUDs

Standard scripts leave prominent console windows open on your display. Using the VBS script host wrapper allows this process to execute silently in the background, utilizing decoupled PowerShell sub-processes to issue clean Windows desktop notification banners during state transitions.
